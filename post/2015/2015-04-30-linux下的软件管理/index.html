Linux下的软件管理

Published on Apr 30, 2015 ,1 min reading time

Linux

软件包分类

源码包

  • 开源，修改源码，选择功能，定制软件
  • 稳定，效率高，卸载方便
  • 安装步骤多，容易出错，耗时长

二进制包

- 包管理简单，速度快
- 闭源，效率低
- 依赖性

RPM包

命名规则

例如: heart-2.1.4-12.el6.x86_64.rpm

依赖性

  • 树形依赖： a->b->c
  • 环形依赖： a->b->c->a
  • 模块依赖： http:www.rpmfind.net

rpm命令管理

- 包名：未曾安装，使用`包全名`及路径；已安装，使用`包名`(/var/lib/rpm/)
- 命令：rpm -ivh [package full name]  
  -i(install)  //安装  
  -v(verbose)  //详细信息  
  -h(hash)  //进度  
  -Uvh(upgrade) //升级  
  -e(erase) //卸载  
  -p(package) //查看未安装的包
  -qa(query，all) //是否安装  
  -qip //包的信息  
  -ql //包安装位置  
  -qf //查询文件属于哪个包  
  -qR  //查看依赖

安装路径

|路径|文件类型|
|:—–|——:|
|/etc/|配置文件|
|/usr/bin|可执行命令|
|/usr/lib|函数库|
|/usr/share/doc|使用手册|
|/usr/share/man|帮助文档|

包校验

rpm -V [package name]  
验证内容无变更，显示为空

|位置|符号|含义|
|—|—|—|
|0|S|文件大小|
|1|M|文件权限|
|2|5|MD5校验和|
|3|D|主从代码|
|4|L|文件路径|
|5|U|所属者|
|6|G|所属组|
|7|T|修改时间|

|字符|文件类型|
|—|—|
|c|配置文件|
|d|普通文档|
|g|ghost文件|
|L|授权文件|
|r|描述文件|

  • RPM包文件提取：rpm2cpio [package name] | cpio -idv 绝对路径

yum命令管理

yum主要功能是更方便的添加/删除/更新RPM包，安装形式为在线安装。降低安装软件的复杂度，所有软件包都在官方在线服务器(也可以使用光盘安装)，可自动解决依赖问题。

yum源文件

默认只有Base.repo生效，其他三个文件不生效

|参数|作用| |:—-|—-:| |[base]/[update]|说明作用| |name|包名| |baseurl,mirrorlist|主站点，镜像站点| |enable|生效值(1)，默认效果视文件而定| |gpgcheck|数字验证(1)| |gpgkey|ca证书文件地址|

  • 光盘安装: 先使在线安装的源文件Base.repo失效,需要使用Media.repo,然后修改该文件中的地址

yum命令

yum list  列出可用软件列表
yum search keyword  搜索服务器和`keyword`相关的包
yum -y install name  安装
yum -y update name  更新
yum -y remove name  卸载

yum grouplist 列出所有可用软件组列表
yum groupinstall grpname 安装软件组
yum groupremove grpname 卸载软件组

源码包

源码包和rpm包的区别：

  • 安装前，参照本文1.1
  • 安装后，参照下表 |RPM默认安装目录|说明| |:—–|—–:| |/etc|配置文件| |/usr/bin|可执行命令| |/usr/lib|函数库| |/usr/share/doc|使用手册| |/usr/share/man|帮助文件|
